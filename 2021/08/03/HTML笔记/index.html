HTML

概念

是最基础的网页开发语言

Hyper Text Markup Language 超文本标记语言

  • 超文本:
    • 超文本是用超链接的方法，将各种不同空间的文字信息组织在一起的网状文本.
  • 标记语言:
    • 由标签构成的语言。<标签名称> 如 html，xml
    • 标记语言不是编程语言

快速入门

语法

  1. html 文档后缀名 .html 或者 .htm

  2. 标签分为

    1. 围堵标签：有开始标签和结束标签。如 <html></html>
    2. 自闭合标签：开始标签和结束标签在一起。如 <br/>

  3. 标签可以嵌套：

需要正确嵌套，不能你中有我，我中有你

  • 错误：<a><b></a></b>

  • 正确：<a><b></b></a>

  2. html 的标签不区分大小写，但是建议使用小写。

文件标签

构成 html 最基 本的标签

  • html : html 文档的根标签

和文本有关的标签

    属性：

    1. 数值：width='20' 数值的单位默认是 px(像素)
    2. 数值%：占比相对于父元素的比例

链接标签

表单标签

概念

用于采集用户输入的数据的。用于和服务器进行交互。

<form></form> : 用于定义表单的。可以定义一个范围，范围代表采集用户数据的范围

  • 属性：
    • action : 指定提交数据的 URL
    • method : 指定提交方式，一共有七种，常用的有两种
      • get：
        • 请求参数会在地址栏中显示。会封装到请求行中
        • 请求参数大小是有限制的
        • 不太安全
      • post：
        • 请求参数不会再地址栏中显示。会封装在请求体中
        • 请求参数的大小没有限制
        • 较为安全

表单项中的数据要想被提交，必须指定其 name 属性

文档结构和基本语法

XHTML

The Extensible HyperText Markup Language => 可扩展标识语言

基本结构

XHTML 是 HTML 向 XML 过渡的标识语言

基本语法

  1. 在文档的开头必须定义文档类型
  2. 在根元素中应声明命名空间，即设置 xmlns 属性
  3. 所有的标签都必须是闭合的
  4. 所有元素和属性都必须小写
  5. 所有的属性都必须用引号（””）括起来
  6. 所有标签都必须合理嵌套
  7. 所有属性都必须被赋值，没有值的属性就用自身来赋值 <td nowrap="nowrap"/>
  8. 所有特殊符号都用编码表示
  9. 不要在注释内容中使用“–”，只能出现在注释开头和结束
  10. 废除了 name 属性，而使用 id 属性作为统一的名称

文档类型 DOCTYPE

  • 声明文档的类型，且必须位于代码的第一行

  • 不需要向 XHTML 那么复杂，<!DOCTYPE html> 即可

  • 会触发浏览器以标准模式显示页面

    浏览器显示模式：

    • 怪异模式 : Quirks
    • 近标准模式 : Alomst Standards
    • 标准模式（非怪异模式）: Standards

HTML5 属性

表单属性

  • autofocus ：让元素在页面打开时自动获得焦点

    • input(type=”text”)、select、textarea、button

  • placeholder ：对用户的输入进行提示

    • input(type=”text”)、textarea

  • form ：声明它属于哪个表单，然后就可以将它放在页面的任何位置，而不一定是表单之内

    • input、output、select、textarea、button、fieldset

  • required ：在用户提交时进行检查，检查该元素内一定要有输入内容

    • input(type=”text”)、textarea

  • 为 input 元素增加 autocomplete、min、max、multiple(允许多文件上传)、pattern 和 step 属性

  • 为 input 和 button 元素增加 formaction、formenctype、formmethod、formnovalidate 与 formtarget 属性

    重载 form 元素的 action、enctype、method、novalidate 与 target 属性

  • 为 fieldset 元素增加了 disabled 属性

  • novalidate ： 取消提交时进行的有关检查，表单可以被无条件地提交

    • input button from

链接属性

  • media ：目标 URL 是为什么类型的媒介/设备进行优化的，只能在 href 属性存在时使用
    • a area
  • 为 area 元素增加了 hreflang 和 rel 属性，以保持与 a 元素、link 元素的一致
  • 为 link 元素增加了新属性 sizes，该属性可以与 icon 元素结合使用(通过 rel 属性)，该属性指定关联图标(icon 元素)的大小
  • 为 base 元素增加了 target 属性，主要目的是保持与 a 元素的一致性

其他属性

  • reversed ：ol 元素 列表项倒序显示
  • 为 meta 增加了 charset 属性
  • 为 menu 增加了两个新的属性 type 和 label
    • label ：为菜单定义一个可见的标注
    • type ：让菜单可以上下文菜单、工具条与列表菜单三种形式出现
  • scoped ：style 元素增加，用来规定样式的作用范围
  • async ：script 元素增加，定义脚本是否异步执行
  • manifest ：html 元素增加，开发离线 Web 应用程序时它与 API 结合使用，定义一个 URL，在这个 URL 上描述文档的缓存信息
  • 为 iframe 增加三个属性 sandbox、seamless 与 srcdoc，用来提高页面安全性，防止不信任的 Web 页面执行某些操作

HTML5 全局属性

contenteditable 在线编辑

允许用户可以在线编辑元素的内容，布尔值属性

  • true ：允许编辑
  • false ：不允许编辑
  • inherit ：未指定 true 或 false 时，由 inherit 状态决定，看父元素是否可编辑

想要保存编辑后的内容，使用 innerHTML 获取

所有主流浏览器都支持 contenteditable 属性

在 JavaScript 中，元素还有一个 isContentEditable 属性，当元素可编辑时，该属性值为 true，反之则为 false

contentmenu 右击菜单

用于定义 div 元素的上下文菜单，在用户右键单击元素时出现

目前只有 Firefox 支持 contentmenu 属性

data-* 自定义属性

使用 data-* 可以自定义用户数据

  • 属性名：不应该包含任何大写字母，在前缀“data-”之后必须至少有一个字符

    • 多个单词使用：- 连接

    • 脚本中获取或访问属性使用 dataset.对应小驼峰式

  • 属性值：可以是任意字符串

当浏览器（用户代理）解析时，会完全忽略前缀为“data-”的自定义属性

所有主流浏览器都支持 自定义属性

draggable 可被拖动

定义元素是否可以被拖动，但不是元素位置改变

true、false、auto(定义使用浏览器的默认行为)

此属性常被用在拖放操作中

dropzone 拖动操作

定义在元素上拖动数据时，是否复制、移动或链接被拖动数据

  • copy ：会产生被拖动数据的副本
  • move ：会导致被拖动数据被移动到新位置
  • link ：会产生执行原始数据的链接

但是，目前主流浏览器都不支持 dropzone 属性

hidden 隐藏

所有元素都包含一个 hidden 属性，设置元素的可见状态，true 不可见，false 可见

除了 IE 所有主流浏览器都支持 hidden 属性

spellcheck 拼写 语法检查

定义是否对元素进行拼写和语法检查，可以对以下内容进行拼写检查：

  • input 元素中的文本值（非密码）
  • textarea 元素中的文本
  • 可编辑元素中的文本
