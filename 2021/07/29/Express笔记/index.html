四、路由

扩展：HTTP 请求类型
OPTIONS、GET、HEAD、POST、PUT、DELETE、TRACEa

简介

  • 什么是路由：就是网址
  • 什么是路由参数：就是网址参数
  • 在 PHP 中作用：声明交给哪个控制器的哪个方法处理
  • 在 NodeJS 中作用：交给指定匿名函数处理

语法

**普通语法**：app.HTTP 请求类型（请求路径，回调函数）

发送**GET请求：app.get**（请求路径，回调函数）

发送**POST请求：app.post**（请求路径，回调函数）

发送**任意请求：app.all**（请求路径，回调函数）

**特殊语法：app.use**（请求路径，回调函数）

区别 1：use 匹配任意类型请求

区别 2：**use 非完全匹配（ps. 只需要url 前面匹配**请求路径即可匹配）

**路由参数：app.HTTP 请求类型**（请求路径/:参数 1/…/:参数 n，回调函数）

八、Express 框架

8.1 中间件

概念

HTTP 请求中的过滤层（HTTP 请求交给方法处理前，先交给另一个方法过滤）

  • PHP 中：BaseController
  • NodeJS 中：app.use()
应用场景

判断登录状态、检测 RBAC 权限、错误捕捉等

8.3 路由中间件

  1. 创建 day3 目录

  2. app.js 中：

    中间件匹配成功后，不会继续往下寻找，需要使用 next()继续访问

    使用next()函数需要添加 next 形参

    function (req, res, next) {}

8.4 错误处理

中间件灵活使用
  • 形参可以不写，但顺序不能改变
  • ‘/‘ 可以省略不写，与写上等价
  • next 如果不传递参数 只匹配下一个 URL；next 传递 err 则匹配第一个含有 err 的中间件

九、会话技术

9.1 cookie

手册

https://www.npmjs.com/package/cookie-parser

语法

  1. 安装 cookie 模块，语法：npm install cookie-parser

  2. 引入 cookie 模块，语法：var cookieParser = require('cookie-parser')

  3. 初始化 cookie，语法：app.use(cookieParser())

                        相当于：`app.user('/', cookieParser())`

    作用：http 传入的 cookie 进行解析后赋值给 req.cookies

    其次：cookieParser 加参数，并且设置是 signed=true 则加密存储（推荐）

设置

res.cookie(键, 值, {signed: 是否加密true-是,false-否默认,maxAge: 时间/毫秒})

获取（未加密）：req.cookies

获取（ 加密）：req.signedCookies

清除：res.clearCookie(键)

十、总结

如何展示静态页面

在 PHP 中
  1. 确定路由
  2. 根据路由创建控制器方法 -> 加载视图 display 或 view 函数
  3. 创建视图 -> 修改静态资源路径
在 NodeJS 中
  1. 定义路由 -> 通过 render 加载视图传递数据
  2. 创建视图 -> 修改静态资源路径

如何数据处理

  1. 修改表单属性值
  2. 接受数据
  3. 逻辑数据
  4. 数据库操作
  5. 判断跳转

如何显示列表

  1. 控制器方法获取数据 -> 传递给视图
  2. 调用模板标签循环显示

如何实现 增删改

  1. 修改静态页面（前：修改表单 action name 后：修改 a 标签 id）
  2. 数据处理（1-接受数据，2-数据过滤，3-入库，4-判断跳转）
