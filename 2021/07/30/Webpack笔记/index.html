使用 Webpack

webpack

当前 Web 开发 面临的困境：

  • 文件以来关系错综复杂
  • 静态资源请求效率低
  • 模块化支持不友好
  • 浏览器对高级 JavaScript 特性兼容程度较低

webpack 概述

webpack 是一个流行的前端项目构建工具（打包工具），可以解决当前 web 开发中所面临的困境。

提供了友好的模块化支持，以及代码压缩混淆、处理 js 兼容问题、性能优化等强大的功能，提高了开发效率和项目的可维护性。

webpack 中的加载器

  1. webpack 默认只能打包处理 .js 结尾的文件，处理不了其他后缀的文件
  2. 代码中包含了其他文件，因此 webpack 默认处理不了
  3. 会查找 webpack.config.js 这个配置文件，看 module.rules 数组中，是否配置了对应的 loader 加载器
  4. 找到后，将文件转交给最后一个 loader 进行处理
  5. 处理完毕后，会把处理的结果转交给上一个 loader
  6. 到最后，前面没有下一个 loader 了，就把最后的结果转交给了 webpack
  7. webpack 将结果合并到 /dist/bundle.js 中，最终生成打包好的文件
