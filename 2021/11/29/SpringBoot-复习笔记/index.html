在 yml 语法中，相同缩进代表同一个级别，使用键值对语法：

dependency

dependency 标签：用来导入依赖

  • artifactId：用来导入项目（jar 包、各种场景启动器）
  • groupId：这些项目的包名
  • version：版本号
  • type：该依赖的项目类型，jar、war…
  • scope：该依赖的作用范围
    • complie：默认值，用于编译、打包、测试、运行阶段
    • provided：不用于打包
    • runtime：不用于编译
    • test：只用于测试
  • exclusion：排除依赖项

注解

@SpringBootApplication

标明该注解的类为主启动类或者主配置类，作为 springboot 程序的入口

SpringApplication.run(MySpringBootApplication.class) 代表 SpringBoot 的启动类，参数为 SpringBoot 启动类的字节码对象。

@ComponentScan

扫描 @SpringBootApplication 下的 @Controller、@Repositroy、@Service、@Component

@RequestMapping

其中还有 @GetMapping、@PostMapping、@PutMapping、@DeleteMapping、@PatchMapping，就相当于：

更新使用 @PutMapping 更合适，添加更偏向于使用 @PostMapping

@RequestBody

请求参数在响应体中：

@RequestParam

限定 HTTP 参数到 Controller 方法的映射关系。

请求参数在请求域中：?userId=2&userAge=25

@PathVariable

从 URL 中提取参数。

绑定参数和请求路径中的参数，接收请求路径中占位符的值

@ResponseBody

将返回的数据放入响应体中，默认格式为 json

@RestController 等同于 @Contoller + @ResponseBody

@Configuration

标注该注解的类是 springboot 的配置类

在配置类中才可以进行 bean 注入

@Mapper

标注该注解的接口是 mybatis 的接口，自动注入到 IOC 容器中

@AutoWired

自动获取 ICO 容器中的组件、实例

一般将它写在字段前或者 setter 方法前面，作为 bean 注入时的使用。

@ModelAttribute

通常作用在 Controller 的某个方法上，会首先被调用，并将方法结果作为 Model 的属性，然后再调用对应的 Controller 处理方法。

IOC

控制反转，是一种思想，意味着将设计好的对象交给 spring 容器管理，而不是直接 new 对象进行控制，spring 容器称为 ioc 容器。

DI 指依赖注入，实际上就是 IOC 的实现。

注入方式

  1. 引入 xml 注入；

  2. 注解注入：

    • @Configuration + @Bean 注入；
    • @service、@Controller、@Mapper、@Component、@Repository 注入 + @ComponentScan 扫描。

  3. 获取 bean：

    • @AutoWired

    • @Resource

    • bean

    • 主启动类直接中获取。

实现流程

此小节来自 娃宇翔（QQ：985391895）！

  1. 声明一个切面类，并用 @Component 和 @Aspect 修饰

  2. 用 @Pointcut 注释声明一个切点，告诉切面谁是他的服务对象（此注释修饰的方法的方法体为空，不需要写功能，可以理解为切点对象的一个代理对象方法）

  3. 在对应的方法前用对应的通知类型注释修饰，将对应的方法声明成一个且切面功能，为了切面服务
