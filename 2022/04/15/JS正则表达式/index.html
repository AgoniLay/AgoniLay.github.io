正则表达式

用于匹配规律规则的表达式，通常被用来检索、替换那些符合某个模式（规则的文本）。

构建

字面量创建：

构造函数创建：

修饰符

  • i：ignoreCase，匹配忽视大小写；
  • m：multiline，多行匹配；
  • g：global，全局匹配。

属性方法

exec()

匹配字符串和正则表达式：

  • 匹配成功：返回一个数组

    [匹配内容, index: 匹配的起始位置, input: 要匹配的字符串, group: undefined]

  • 匹配失败：返回 null

test()

测试待检测的字符串是否能匹配到，匹配到返回 true，否则返回 false。

compile()

可以编译指定的正则表达式，编译之后的正则表达式执行速度将会提升（在多次被调用时效果会较为明显）.

lastIndex

未设置全局匹配时，返回值均为 0；

设置全局匹配后，匹配到对应字符 lastIndex 指向该字符的结束位置，在执行 exec 返回 null 时 lastIndex 归零；

元字符

字符 匹配
字母和数字字符 自身
\o Null 字符
\t 制表符 Tab
\n 换行符
\v 垂直制表符
\f 换页符
\r 回车符
字符类 含义
. 匹配 换行符\n 和 回车符\r 之外的任何单个字符，等效于 [^\n\r]
\d 匹配一个数字字符，等效于 [0-9]
\D [^0-9]
\w 匹配包括下划线的任何单个字符，等效于 [a-zA-Z0-9]
\W [^a-zA-Z0-9]
\s 匹配任何 Unicode 空白字符，包括空格、制表符、换页符和换行符，等效于 [\f\t\n\r]
\S [^\f\t\n\r]
  • []：匹配其中的某一个字符；
  • ^：
    1. 反义字符：[^abc]；匹配除了 abc 之外的字符；
    2. 边界符：表示匹配开始的字符；
  • $：表示匹配结束位置的字符；
    • ^ 和 $ 共同使用表示精确匹配，只有匹配的字符完全相同才匹配成功；
字符 含义
* ≥ 0 次
+ ≥ 1 次
? 0 或 1 次
{n} n 次
{n,} ≥ n 次
{n, m} n 到 m 次
  • 贪婪模式：默认为贪婪模式，尽可能匹配更多的
  • 非贪婪模式：尽可能匹配少的，在数量前加 ? 改变为非贪婪模式

String 方法

search()

和 exec() 类似，有则返回第一次出现的索引，否则返回 -1

match()

不加全局匹配时和 exec 一样，加了之后以数组形式返回查找到的所有的字符。

split()

按照指定规则切割字符串

replace()

替换满足正则表达式的内容，全局模式替换所有匹配的，非全局模式只替换第一个。
