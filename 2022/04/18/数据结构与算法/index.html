数据结构与算法

程序设计 = 数据结构 + 算法；数据 = 符号

  • 数据：指可以输入到计算机中并且能够被计算机识别和处理的符号；
  • 数据元素：数据的基本单位，也成为节点或者记录；
  • 数据对象：相同特性的数据元素的集合，是一个数据的子集；
  • 数据项：独立含义的数据的最小单位

数据的目的是存储，存储的目的是：后期再利用

复杂度

时间复杂度

Big O notation：

  • **O(1)**：Constant Complexity 常数复杂度

  • **O(log n)**：Logarithmic Complexity 对数复杂度

  • **O(n)**：Linear Complexity 线性时间复杂度

  • **O(n2)**：N square Complexity 平方

  • **O(n3)**：N square Complexity 立方

  • **O(2n)**：Exponential Growth 指数

  • **O(n!)**：Factorial 阶乘

只看最高复杂度的运算。

主定理(Master Theorem)

主定理适用于求解如下递归式算法的时间复杂度：

  • n：问题规模大小
  • a：原问题的子问题个数
  • n/b：是每个子问题的大小，假设每个子问题有相同的规模大小
  • f(n)：是将原问题分解成子问题和将子问题的节合并成原问题的解的时间

上式的复杂度分析主要如下列三种情况（简化版本）：

  1. 若 f(n) > O(nlogba) 且满足 a f(n/b) ≤ c f(n)，则 T(n) = O(f(n))；
  2. 若 f(n) < O(nlogba)，则 T(n) = O(nlogba)；
  3. 若相等，则 T(n) = O(nlogbalogk+1n)

常用算法的应用：

算法 关系表达式 时间复杂度 注释
二分查找 T(n) = T(n/2) + O(1) O(log n) c = logb a, a = 2, b = 2, c = 0, k = 0[5]
二叉树遍历 T(n) = 2 T(n/2) + O(1) O(n) c < logb a, a = 2, b = 2, c = 0[5]
有序的二维矩阵 T(n) = 2 T(n/2) + O(log n) O(n)
归并排序 T(n) = 2 T(n/2) + O(n) O(n log n) c = logb a, a = 2, b = 2, c = 1, k = 0

数据结构

数组

数组（Array）的随机查找一个元素的时间复杂度是 O(1) 的，因为内存中给每一个元素分配的固定的地址供随机访问索引：

但在插入和删除元素时时间复杂度是 O(n) 的，因为该元素之后的所有元素的未知均需要发生改变：

如果是往数组最后一个元素插入或删除，则复杂度是 O(1) 的，但如果是首个元素，则复杂度是最差的 O(n)，平均的时间复杂度是 O(n/2)，也就是 O(n) 的。

链表

单链表（Linked List）包括一个值和一个指向下一个位置的指针。

对链表进行添加删除操作只需要两次即可，也就是常数级别的时间复杂度 O(1)：

但在需要查询元素时，需要从头指针依次进行查找，复杂度与数组添加删除类似，为 O(n)。
